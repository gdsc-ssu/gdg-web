# Solution Challenge 2025 참여 후기

안녕하세요, GDGoC Soongsil입니다!

올해 Google Solution Challenge 2025에 처음으로 팀을 꾸려 참여했습니다. 약 두 달간의 여정을 돌아보며 느낀 점을 공유합니다.

## Solution Challenge란?

Google Developer Groups on Campus(GDGoC) 주관으로 매년 열리는 글로벌 해커톤으로, **UN이 정한 17가지 지속가능발전목표(SDGs)** 중 하나를 기술로 해결하는 프로젝트를 만드는 대회입니다.

## 우리 팀의 아이디어

저희 팀은 SDG 3번 **"건강과 웰빙"** 을 주제로, 1인 가구 청년들의 정신건강을 돕는 앱을 개발했습니다.

주요 기능은 다음과 같습니다.

- **감정 일기** — 매일 짧게 기록하면 AI가 감정 패턴을 분석
- **커뮤니티** — 비슷한 고민을 가진 사람들과 익명으로 연결
- **전문가 연결** — 필요 시 심리 상담사와 빠르게 매칭

## 기술 스택

| 역할 | 기술 |
|------|------|
| Android | Kotlin, Jetpack Compose |
| Backend | Spring Boot, PostgreSQL |
| AI | Gemini API |
| 인프라 | Google Cloud Run |

## 개발하면서 어려웠던 점

### 1. Gemini API 프롬프트 설계

감정 분석의 품질이 프롬프트 설계에 크게 좌우됐습니다. 초기에는 단순히 "이 텍스트의 감정을 분석해줘" 수준이었는데, 출력이 너무 일관성이 없었습니다.

여러 번의 반복 끝에 **Few-shot 예시**를 함께 전달하는 방식으로 개선했고, 결과 품질이 눈에 띄게 좋아졌습니다.

### 2. 팀원 간 Git 협업

각자 개발 경험이 달라서 초반에 충돌이 잦았습니다. 브랜치 전략을 `feature/기능명` 형태로 통일하고, PR 리뷰를 필수화한 뒤 훨씬 수월해졌습니다.

## 결과와 배운 점

Top 100에는 들지 못했지만, 아이디어를 실제 동작하는 앱으로 만들어 낸 경험 자체가 큰 수확이었습니다.

> "완벽한 아이디어보다 빠르게 만들고, 빠르게 부수는 게 낫다"

두 달 동안 매주 미팅을 하면서 팀워크와 커뮤니케이션의 중요성을 몸으로 배웠습니다. 내년에는 꼭 Top 100을 목표로 다시 도전할 예정입니다!

GDGoC Soongsil에서는 매년 Solution Challenge 참가자를 모집합니다. 기술에 관심 있고, 함께 문제를 풀어가고 싶은 분이라면 누구든 환영합니다.
